Scrapboxでどの程度お礼を書けばいいかがわからない

  • 井戸端でも相談した
    • /villagepump/お礼やいいねはどう書けばいいか
    • されても嫌な人はいない
  • 何に困っている？
    • お礼書こうかな、どうしようかな、どういう文面にしようかな ← こういう判断が面倒くさい
    • お礼書かなかったとして相手がむっとするのではないか…… ← こういう思考も出てきて面倒くさい
      • アイツはお礼言わないからもう二度と助けん
    • いいねマシーンはそれはそれでノイズになるのではって気もする
      • Scrapboxは情報をやりとりする場所なので「あざっす」みたいな行はゴミでしかない
        • 感謝行
      • 井戸端のような雑談エリアでは別に構わない
        • でも井戸端にも情報をやりとりしてる雰囲気のページがあって、そこであざっす使うのはなーとも思う
  • 例
    • /villagepump/@sta
      • ここは毎回使えばいい
      • 特に「staさん、～～のページに～～とありますけど、それ～～でいけます」的なアドバイス
      • 実際普通にありがたいので自然とあざっすしちゃう
    • /villagepump/alias tool#600972605b98c20000470edd
      • ここが微妙
      • alias は shell の機能だよ
      • おお、そうだったのか、あざっす……と書きたいところだが、感謝行を書くのがちょっとはばかられる
  • もうお礼カード使っちゃうか？ｗ
    • 個人ページにこんなことを書いておく
      • 私は自分の書いた文章に対する返信すべてを読んでいます（読んでいると思います）
      • いただいたアドバイスなども目を通しており、すべてについて感謝をしています
      • 問題がある場合は返信をします
      • つまり、返信がない＝問題がない＆感謝している、と捉えて構いません
    • これで「staはデフォルトで感謝する人間である」にできる


bra

  • 気にしぃが強いんだろうなぁと思う
    • チャットとかでも「返事来ねえな」「遅いな」「もしかして嫌われているのでは……」 すぐこうなるｗ
      • 豆腐メンタル
  • パターン別に「このときは要らない」「このときは書く」を蓄えた方がいいかも
    • たとえば「staが書いたことが明らかでない場合」は書かなくていい、とか
      • 上記 alias の例では、staが書いた文脈ではないので書かなくていい
      • 返信していない = alias is a part of shell functions のツッコミが正しいことの証左、とみなせる
  • リテラシークラスターだなぁと思った
    • こういうことを気にするのは僕が大企業に染まってるからなのかもしれない
      • いいね一つに神経使う世界だからねぇ……
      • 僕はいいねマシーンだったりするけどｗ
    • たぶん井戸端の他の人はそんなに気にしてない
  • 僕の場合はどうなん？お礼とか気にする？
    • 全く気にしない
    • 僕は好き勝手に新規したりツッコミ入れてりしてるだけ
    • :sta:だったら僕も気にしなくていいのでは？
